### 4.3 冷启动解决方案

| 冷启动类型       | 解决策略                                       |
| ---------------- | ---------------------------------------------- |
| **新用户冷启动** | 引导填写风格偏好问卷 → 内容推荐 + 热门商品推荐 |
| **新商品冷启动** | 基于商品属性特征匹配 → 内容推荐先行            |
| **数据稀疏**     | 矩阵分解 + 深度学习弥补协同过滤的稀疏性问题    |

CREATE TABLE users (
    user_id   INT PRIMARY KEY AUTO_INCREMENT,

    password  VARCHAR(255) NOT NULL,

    style_pref JSON,          -- 风格偏好标签
    size_info  JSON,          -- 尺码信息（身高/体重/胸围等）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 商品表
CREATE TABLE products (
    product_id  INT PRIMARY KEY AUTO_INCREMENT,
    title       VARCHAR(200),
    category    VARCHAR(50),
    style       VARCHAR(50),
    price       DECIMAL(10,2),
    sales_count INT DEFAULT 0,
    images      JSON,          -- 图片URL数组
    attributes  JSON,          -- 尺码/颜色/材质
    embedding   BLOB           -- 预计算的特征向量（可选）
);

-- 用户行为日志表（推荐算法的核心输入）
CREATE TABLE user_behaviors (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id     INT,
    product_id  INT,
    action_type ENUM('view','collect','cart','purchase'),
    duration    INT,           -- 停留时长（秒）
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user (user_id),
    INDEX idx_product (product_id)

CREATE TABLE recommendations (
    user_id    INT,
    product_id INT,
    score      FLOAT,
    algo_type  VARCHAR(20),   -- 'cf'/'content'/'deep'/'hybrid'
    created_at TIMESTAMP
);

> 💡 **关键评价指标：**

> - **NDCG@K** = 归一化折损累积增益（衡量排序质量）

## 七、可能踩的坑 & 应对建议

| 风险点                         | 应对策略                                               |
| ------------------------------ | ------------------------------------------------------ |
| 淘宝反爬导致数据采集失败       | 使用学术公开数据集（Amazon Fashion、Kaggle服装数据集） |
