# 服装推荐系统：全模块技术深度解析 (答辩专用)

本手册针对答辩中可能遇到的“模块功能实现细节”进行深度拆解，覆盖全站每一个核心业务节点。

---

## 一、 后端功能模块全拆解 (Backend Deep Dive)

后端基于 **Flask** 框架，采用模块化蓝图 (Blueprint) 设计。

### 1. 用户认证模块 (Auth Module)
-   **代码文件**: `backend/api/auth.py`
-   **核心功能**:
    -   **注册/登录**: 通过 `werkzeug.security` 实现密码的哈希加密存储（哈希+盐），确保数据库安全。
    -   **鉴权中心**: 登录后签发带有 `sub` (user_id) 的 JWT 令牌。
-   **技术亮点**: 实现了**无状态鉴权**，服务器不维护 Session，极大提升了系统的水平扩展能力。

### 2. 用户行为引擎 (Behavior Module)
-   **代码文件**: `backend/api/behavior.py`, `backend/models/behavior.py`
-   **核心功能**:
    -   **多维埋点**: 支持 `view` (浏览)、`collect` (收藏)、`cart` (加购)、`purchase` (购买) 四种原子操作的实时上报。
    -   **行为溯源**: 记录 `product_id` 和时间戳，为个性化算法提供最原始的偏好数据流。
-   **技术亮点**: 接口支持**冥等性**校验，防止因网络波动造成的重复行为统计。

### 3. 个性化推荐引擎 (Recommend Module)
-   **代码文件**: `backend/api/recommend.py`
-   **核心功能**:
    -   **热门推荐**: 针对未登录或新用户，通过 SQL 聚合统计 `UserBehavior` 表，返回全站热度最高的商品。
    -   **个性化精选**: 根据 `user_id` 从 `recommendations`（结果表）中提取混合模型计算出的 Top-N。
    -   **相似推荐**: 在详情页调用，基于当前商品的 `style` 或 `category` 属性进行内容关联。
-   **技术亮点**: 采用**结果预缓存模式**，推荐响应时间由秒级（实时计算）降低至 10ms 以内。

### 4. 订单与支付模拟 (Order Module)
-   **代码文件**: `backend/api/order.py`, `backend/models/order.py`
-   **核心功能**:
    -   **订单状态机**: 维护 `待支付`、`已支付`、`已取消` 三种状态自动流转。
    -   **结算汇总**: 自动计算优惠后的实付金额，并校验商品库存完整性。
-   **技术亮点**: 实现了**事务锁定**，在创建订单的同时预扣库存，防止超卖现象。

## 二、 前端功能模块全拆解 (Frontend Deep Dive)

前端基于 **Vue 3** + **Element Plus**，采用典型的淘宝组件化布局。

### 1. 淘宝级首页系统 (Home Module)
-   **代码文件**: `frontend/src/views/HomeView.vue`
-   **核心功能**:
    -   **层级导航**: 侧边栏垂直分类及二级菜单悬浮展示。
    -   **多路召回展示**: 同时呈现“今日热点”与“猜你喜欢”两个瀑布流区块。
    -   **冷启动引导**: 为新用户提供 `StyleTest` 弹窗，引导其进行兴趣打标。

### 2. 商品详情与交互 (Product Module)
-   **代码文件**: `frontend/src/views/ProductDetailView.vue`
-   **核心功能**:
    -   **SKU 选择器**: 动态联动颜色、尺码。只有选中全部属性后才允许“加入购物车”。
    -   **交互回馈**: 动作完成后的飞入购物车动画效果及 Toast 提示。
    -   **详情富文本**: 图片瀑布流展示服装细节及尺寸对照表。

### 3. 全球购物车与结算 (Cart & Checkout)
-   **代码文件**: `frontend/src/store/cart.js`, `frontend/src/views/CheckoutView.vue`
-   **核心功能**:
    -   **状态持久化**: 购物车数据实时同步至 `localStorage`，刷新不丢失。
    -   **地址联想**: 结算页支持最近收货地址记忆与快捷选择。
    -   **虚拟支付流程**: 模拟真实的第三方支付跳转与成功/失败反馈。

### 4. 数字化管理大屏 (Admin Dashboard)
-   **代码文件**: `frontend/src/views/admin/DashboardView.vue`
-   **核心功能**:
    -   **算法大比武**: 直观对比各版本算法（SVD vs DeepFM）的离线评测数据。
    -   **生意参谋**: 实时监控全站的商品售卖占比及 7 日用户增长趋势。
-   **技术亮点**: 采用 **ECharts** 异步渲染机制，大屏数据按需加载，丝滑流畅。

## 三、 答辩进阶技巧：如何体现“复杂性”

1.  **数据流视角**:
    -   从用户产生一条 `behavior` 到后端入库，再由 Notebook 读取进行 `Hybrid` 训练，最后生成结果写回 MySQL 供 API 调用。这种**数据闭环**体现了生产环境的思维。
2.  **安全性视角**:
    -   后端通过 `jwt_required` 保护所有敏感接口；前端通过 `router.beforeEach` 阻止非法访问；数据库通过 ORM 机制防注入。
3.  **算法鲁棒性视角**:
    -   介绍了由 **User/Item-CF**（解决相似性）、**SVD**（解决稀疏性）和 **DeepFM**（解决特征交叉）构成的三叉戟模型，极大降低了由于单一模型失效带来的推荐偏差。
4.  **工程化细节**:
    -   提到的“库存预扣机制”、“无刷新的 Pinia 状态同步”、“ECharts 的响应式布局”等都是从 MVP (最小可行性产品) 向“真正可商用软件”迈进的体现。
